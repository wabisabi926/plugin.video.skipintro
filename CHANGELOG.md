# 更新日志

## v1.3.7 (2026-05-19)

### 代码质量优化

#### 1. 统一通知函数
- 将 `service.py` 和 `default.py` 中重复的 `show_notification` 函数统一到 `common.py`
- 标准化函数签名：`show_notification(title, message, duration=3000)`
- 消除代码重复，便于维护

#### 2. 完善异常处理
- 新增 `log_exception(e, context)` 函数，提供统一的异常日志格式和完整堆栈跟踪
- 更新关键路径的异常处理，替换简单的 `except Exception: pass`
- 提高调试效率，快速定位问题

#### 3. 添加类型提示
- 添加 `typing` 模块导入
- 为关键函数、类成员和方法添加类型注解
- 提高代码可读性和 IDE 支持

#### 4. 缓存清理机制
- 添加 `cleanup_expired_caches()` 函数自动清理过期缓存
- 添加 `clear_all_caches()` 函数手动清空所有缓存
- 限制 `season_episodes` 缓存最大条目数为 50 条
- 主循环每 5 分钟自动执行一次缓存清理
- 防止内存泄漏，优化内存使用

#### 5. 配置常量集中化
- 集中定义所有魔法数字为常量：
  - `OUTRO_COUNTDOWN_SECONDS = 6.0` - 片尾倒计时秒数
  - `OUTRO_TRIGGER_THRESHOLD_SECONDS = 6.0` - 片尾触发阈值
  - `RETRY_MAX_ATTEMPTS = 10` - 重试最大次数
  - `RETRY_DELAY_MS = 1000` - 重试延迟毫秒数
  - `CACHE_CLEANUP_INTERVAL_SECONDS = 300.0` - 缓存清理间隔
- 使用常量替代魔法数字，便于调整参数

### 文件变更

| 文件 | 修改类型 | 说明 |
|------|----------|------|
| `common.py` | 修改 | 添加统一通知、异常处理、缓存清理、类型提示、配置常量 |
| `service.py` | 修改 | 使用 common.py 中的统一函数和常量 |
| `default.py` | 修改 | 使用 common.py 中的统一函数和异常处理 |
| `addon.xml` | 修改 | 更新版本号至 1.3.7 |
